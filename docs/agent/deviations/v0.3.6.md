## DEV-026 — cascade_depth field added to ClutchDecision and ClutchDecisionMade event payload

**What changed:** `ClutchDecision` gained `cascade_depth: int = 0` (default 0, backward compat). `ClutchDecisionMade` event payload now includes `cascade_depth` and `escalate_to_catalyst` fields. Two construction sites in `cycle_runtime.py` updated.

**Why:** Observability. Without cascade_depth, there's no way to know whether a cycle accepted quickly (first rule) or deep in the cascade (fifth rule). This matters for catalyst escalation logic in Step 2 and for debugging rule ordering.

**Impact on fossic vocabulary:** `ClutchDecisionMade` event shape extended. New fields are additive (existing consumers that don't read them are unaffected). `docs/aseptic/cross-pollination/pass-9.1.md` notifies fossic of the vocabulary extension.

**Scope:** Non-breaking. Default value (0) means any code reading old event payloads without the field will get a missing-key error if it assumes the field exists, but no current consumer reads `cascade_depth` from stored events.

## DEV-027 — composite_floor field added to CycleConfig (non-breaking, default 0.3)

**What changed:** `CycleConfig` dataclass gained `composite_floor: float = 0.3`. `_parse_config` reads it from YAML via `data.get("composite_floor", 0.3)`. `CycleRuntime` reads `self.config.composite_floor` for per-step `consecutive_steps_below_floor` tracking.

**Why:** The `consecutive_steps_below_floor` predicate needs a reference floor per cycle. Making it a cycle-level config field allows per-cycle tuning (e.g., critical cycles use 0.5; exploratory cycles use 0.2). The stop condition `composite_floor_consecutive` continues to use its own `threshold` parameter — two independent concepts with coincidentally similar semantics.

**Impact:** All existing cycle configs that lack `composite_floor` continue to parse and run without modification. `simple.planning.v0.yaml` was not modified. Default value (0.3) matches the value previously hardcoded in stop condition parameters.

## DEV-028 — catalyst_was_invoked ships as always-False placeholder

**What changed:** `catalyst_was_invoked` predicate added to `BUILTIN_PREDICATES` and `BUILTIN_PREDICATE_NAMES`. Always returns `False` in Phase 9 Step 1.

**Why:** Adding the predicate name to the registry now allows cycle configs written for Phase 9 Step 2 to reference it today. Cycle configs are validated against `BUILTIN_PREDICATE_NAMES` at load time; if the name isn't registered, loading fails. By shipping the name in Step 1, Step 2 only needs to wire the implementation — no config-layer changes required.

**Impact:** Any cycle config referencing `catalyst_was_invoked` will load successfully but the predicate will always evaluate to False until Step 2 ships the real implementation.

## DEV-029 — ClutchContext new fields all optional for backward compat

**What changed:** `ClutchContext` gained four new fields: `evaluation: EvaluationPacket | None = None`, `outcome: OutcomeRecord | None = None`, `cycle_state: ClutchCycleState = field(default_factory=ClutchCycleState)`, `cycle_config: CycleConfig | None = None`. All have defaults.

**Why:** 28 existing tests in `test_clutch.py` construct `ClutchContext` directly with keyword arguments matching the original 5-field schema. Making all new fields optional preserves those tests without modification.

**Impact:** The 7 new state-aware predicates gracefully return `False` when their required context field is `None`. This means predicates like `prediction_severe_miss` can be safely registered even in engine configurations where `outcome` is not available (e.g., LLM failure paths in `cycle_runtime.py`).

## DEV-030 — cross-pollination file convention adopted (first use)

**What changed:** `docs/aseptic/cross-pollination/` directory created. `pass-9.1.md` is the first cross-pollination file.

**Why:** `ClutchDecisionMade` event payload extension (`cascade_depth`, `escalate_to_catalyst` now documented as emitted fields) affects the canonical `AGENT_TRACE_VOCABULARY.md` that fossic owns. Aseptic discipline requires notifying the other project. Cerebra has not previously produced a cross-pollination file.

**Convention adopted:** File naming follows `pass-N.M.md` (matching fossic's pattern from prior cross-pollination artifacts). Directory is `docs/aseptic/cross-pollination/` under the Cerebra repo root. This is the canonical location for all future cross-pollination files.
