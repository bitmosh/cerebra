## DEV-019: CycleStarted causation chain restored (D3 fix)

**Spec (retroactive):** `CycleStarted` should causally chain from `SessionOpened` so the full
agent-trace causation tree is traversable across stream boundaries.

**Status in Step 2 (DEV-014):** `CycleStarted` was emitted with `causation_id=None`, making it
the root of the cycle's stream without linkage to the session's `SessionOpened` event.

**Resolution in Step 3:** `SessionManager.open_session()` return type changed from `RuntimeSession`
to `tuple[RuntimeSession, bytes]`. The bytes value is the fossic event_id of `SessionOpened`.
`CycleRuntime.__init__` accepts `opened_event_id: bytes | None = None`. When provided, it is used
as `causation_id` for `CycleStarted`.

**Migration needed:** All callers of `open_session()` must unpack the tuple. Updated:
- `tests/integration/test_session_e2e.py` — 14 callers via `s, _ = manager.open_session(...)`
- `tests/integration/test_cycle_e2e.py` — `_open_session()` helper unpacks tuple
- `cerebra/cli/main.py` — `run-cycle` command unpacks `session, opened_event_id`

**Backward compatibility:** `opened_event_id` is optional in `CycleRuntime.__init__`; passing
`None` (old behavior) still works and produces a root CycleStarted event.

## D1 Blocker — Episode DB write (carried from DEV-017)

**Status:** Still blocked. `memory_records` table has NOT NULL FKs incompatible with cycle
episodes (requires source_id, document_id, chunk_id — not available for LLM-generated content).

**Decision pending:** Three options were posted to #brainstorm:
- Option A: Schema surgery on `memory_records` (make FKs nullable via table recreation)
- Option B (recommended): Separate `cycle_episode_records` table with FK to `runtime_sessions`
- Option C: Fake placeholder rows in sources/documents/chunks

**Current state:** `MemoryWriteFromCycle` events fire with synthetic `record_id = _generate_id("episode")`
but no corresponding DB row. Phase 9 implements the chosen strategy once #brainstorm resolves it.
