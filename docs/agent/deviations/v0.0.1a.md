## Chunker bug location

**Plan said (v0.0.1a brief):** "The bug is in chunk emission, not heading detection. The state machine is tracking the current heading but only emitting one chunk per document at end-of-file... Likely shape of the fix: when the chunker hits a new heading boundary, emit a chunk for the *previous* section."

**Shipped:** The bug was in `_build_sections` in `cerebra/ingest/adapters/markdown.py`, not in `cerebra/ingest/chunking.py`. The chunker correctly iterates `doc.sections` and produces one chunk per section. The problem was that `_build_sections` returned only one section per document (the last heading's content). `chunking.py` was innocent.

**Why:** Read-before-plan identified the exact failure path: the outer condition `if heading_stack or idx == 0` never fired after idx=0 because `heading_stack` was only populated inside the `if idx > 0:` inner guard â€” a circular dependency. The chunker was always being called with a 1-section document, and correctly producing 1 chunk.

**Impact:** Fix is localized to `_build_sections` rewrite. The chunker, pipeline, storage, and tests are unchanged except for the new regression tests.

**Status:** accepted
